Dog API Guide

Summary of Routes

Note: URIs relative to localhost:3000/api

Action HTTP request Description
list GET /dogs Get a list of all the current dogs with name, breed, and id.
find GET /dogs/:id Find the dog associated with that id.
insert POST /dogs Add a new dog.

Get a list of all the dogs

Usage

This sample usage returns an array of dogs, sorted by name, and limited to 3, formatted as JSON. 

  $.ajax({
    method: 'GET',
    data: { max: 3, orderBy: 'name' }
    url: '/api/dogs',
    dataType: 'json'
  })

HTTP Request

GET http://localhost:3000/api/dogs

Parameters

The following table lists the parameters that this query supports. All of the parameters listed below are query parameters.
Parameter Allowed values Required?
max any integer NO
orderBy name, breed NO

Get a specific dog

Usage

This sample usage returns the dog associated with id SyMsy7XHNb, formatted as JSON. If no dog exists with that id, will respond with 404. 

  $.ajax({
    method: 'GET',
    url: '/api/dogs/SyMsy7XHNb',
    dataType: 'json',
  })

HTTP Request

GET http://localhost:3000/api/dogs/:id

Parameters

There are no parameters supported for this endpoint.

Add a new dog

Usage

This sample usage creates a dog with name Noodle and breed Poodle. Incorrectly formatted request bodies will receive a response with status code 400. 

  $.ajax({
    method: 'POST',
    url: '/api/dogs',
    data: JSON.stringify({ name: 'Noodle', breed: 'Poodle'}),
    contentType: 'application/json',
  })

HTTP Request

POST http://localhost:3000/api/dogs

Parameters

There are no parameters supported for this endpoint.
